## Pernah dengar framework [**Next.js**](https://nextjs.org/)?

Saat ini, [Next.js](https://nextjs.org/) menjadi framework fullstack yang sangat populer. [**Next.js**](https://nextjs.org/) adalah framework JavaScript yang dibangun di atas [**React.js**](https://reactjs.org/), dirancang untuk membuat aplikasi web yang cepat dan skalabel.

[Next.js](https://nextjs.org/) sendiri memiliki beberapa metode untuk merender aplikasi, yaitu:

- [**Static Site Generation (SSG)**](https://nextjs.org/docs/basic-features/pages#static-generation): Membuat halaman pada saat build time. Ini ideal untuk konten yang jarang atau tidak berubah, seperti halaman pemasaran atau blog.
- [**Server-Side Rendering (SSR)**](https://nextjs.org/docs/basic-features/pages#server-side-rendering): Menghasilkan halaman dinamis pada setiap permintaan. Cocok untuk konten yang dipersonalisasi atau memerlukan data real-time.
- [**Incremental Static Regeneration (ISR)**](https://nextjs.org/docs/basic-features/data-fetching/incremental-static-regeneration): Kombinasi dari SSR dan SSG, memungkinkan pembaruan konten statis secara bertahap tanpa perlu membangun ulang seluruh situs.
- [**Client-Side Rendering (CSR)**](https://nextjs.org/docs/basic-features/data-fetching/client-side): Merender halaman sepenuhnya di browser menggunakan JavaScript/React, cocok untuk halaman yang sangat interaktif atau membutuhkan pembaruan secara real-time.
- [**Partial Prerendering (PPR)**](https://nextjs.org/docs/advanced-features/partial-hydration): Fitur eksperimental di Next.js yang menggabungkan SSG dan SSR, dengan cara merender konten statis secara langsung, lalu secara bertahap melakukan streaming untuk konten dinamis.

Setiap metode memiliki keunggulan dan kekurangan masing-masing. Penting bagi developer untuk memahami metode-metode rendering ini agar dapat mengoptimalkan aplikasi.
